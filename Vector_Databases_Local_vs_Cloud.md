# Vector Databases: Local vs Cloud

## What is a Vector Database?

A **Vector Database (Vector DB)** stores, indexes, and retrieves vector
embeddings generated by embedding models. Unlike traditional databases,
it performs **semantic similarity search**.

## Workflow

``` text
Documents
   ↓
Embedding Model
   ↓
Dense Vectors
   ↓
Vector Database
   ↓
Similarity Search
   ↓
Relevant Chunks
   ↓
LLM
```

## Local Vector Database

Examples: **FAISS, ChromaDB, Qdrant (Docker)**

**Pros** - Free - Offline - Fast for development - Full control

**Cons** - Harder to deploy - Limited collaboration - Self-managed

## Cloud Vector Database

Examples: **Qdrant Cloud, Pinecone, Weaviate Cloud**

**Pros** - Accessible anywhere - Easy deployment - Scalable - Managed by
provider

**Cons** - Internet required - Free-tier limits - Paid plans for larger
workloads

## Local vs Cloud

  Feature       Local      Cloud
  ------------- ---------- ---------------
  Runs On       Your PC    Remote Server
  Internet      No         Yes
  Best For      Learning   Production
  Scalability   Limited    High

## Examples

### ChromaDB

``` python
vectorstore = Chroma.from_documents(
    documents=docs,
    embedding=embeddings,
    persist_directory="./.chroma"
)
```

### Qdrant Cloud

``` python
client = QdrantClient(
    url=QDRANT_URL,
    api_key=QDRANT_API_KEY
)
```

## Which One Should You Use?

**Local** - Learning - Small projects - Offline development

**Cloud** - Production applications - FastAPI deployments - Multi-user
systems - Team collaboration

## Key Takeaways

-   Vector databases store embeddings instead of plain text.
-   They enable semantic search.
-   FAISS and ChromaDB are excellent for learning.
-   Qdrant Cloud is a strong choice for production-ready RAG
    applications.
